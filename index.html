About Me

저는 기술을 통해 실질적인 문제를 해결하고, 특히 교육 분야에서 긍정적인 변화를 만드는 것에 큰 가치를 둡니다. AI 기술이 개인의 잠재력을 극대화할 수 있는 훌륭한 도구라고 믿으며, 이 기술을 활용해 사용자에게 실질적인 도움을 주는 솔루션을 만드는 데 집중하고 있습니다.

'Studia' 프로젝트는 이러한 저의 철학이 담긴 결과물입니다. 방대한 학습 자료 앞에서 좌절하고, 본인의 이해도를 객관적으로 점검하기 어려웠던 학생들의 '진정한 이해'를 돕고자 AI를 활용한 맞춤형 학습 솔루션을 기획하고 처음부터 끝까지 개발했습니다.

저는 Python과 Node.js를 활용한 백엔드 API 구축부터 HTML/CSS/JS 기반의 직관적인 UI/UX 설계까지, 풀스택 역량을 기반으로 아이디어를 실현하는 과정을 즐깁니다. 앞으로도 생성형 AI와 같은 새로운 기술을 꾸준히 학습하며 사용자에게 실질적인 가치를 제공하는 개발자로 성장하고 싶습니다.

Project Details

문제 정의 (Problem Definition)

학습 과정에서 누구나 방대한 자료를 소화하고 핵심을 파악하는 데 막대한 시간과 노력을 쏟습니다. 하지만 더 큰 문제는, 학습한 내용을 스스로 점검하고 '내가 정말 이것을 이해했는가?'를 객관적으로 확인하기 어렵다는 점이었습니다. 이러한 '진정한 이해'에 대한 갈증과 객관적 이해도 점검의 부재를 해결하는 것이 본 프로젝트의 핵심 목표였습니다. 이를 위해 학습 자료를 기반으로 AI가 핵심을 요약하고, 맞춤형 문제를 생성해 사용자의 객관적인 이해도 점검을 돕는 'Studia'를 개발하게 되었습니다.

아키텍처/설계 (Architecture & Design)

  • Frontend (Client): TypeScript와 React.js를 기반으로 한 싱글 페이지 애플리케이션(SPA)으로 구축했습니다. Tailwind CSS를 사용해 반응형 UI를 구현했으며, React Query를 도입해 서버 상태(요약, 문제 데이터)를 효율적으로 캐싱하고 stale-while-revalidate 전략으로 사용자 경험을 최적화했습니다.
  • Backend (Server): Python의 FastAPI를 채택하여 비동기(Asynchronous) API 서버를 구축했습니다. FastAPI를 통해 AI (LLM) API 연동, PDF/이미지 OCR 처리 등 상대적으로 시간이 오래 걸리는 I/O 작업의 병목 현상을 최소화하고 빠른 응답 속도를 확보했습니다.
  • Deployment (DevOps): Docker를 사용해 FastAPI 백엔드(Python, Tesseract OCR 엔진 등 복잡한 의존성 포함)를 컨테이너화했습니다. 프론트엔드는 Vercel에, 백엔드 컨테이너는 GCP Cloud Run에 배포하여, 트래픽이 있을 때만 동작하는 비용 효율적인 서버리스 아키텍처를 완성했습니다.

핵심 기능 (Core Features)

AI 텍스트 요약

사용자가 PDF, TXT, 사진 등의 학습 자료를 업로드하면, 백엔드 서버에서 PyMuPDF (PDF) 및 Tesseract (OCR)를 통해 텍스트를 추출합니다. 이후 거대 언어 모델(LLM) API와 최적화된 프롬프트 엔지니어링을 통해 방대한 내용의 핵심 개념과 주제를 자동으로 추출하고 간결하게 요약하여 제공합니다.

AI 맞춤형 문제 생성

AI가 1차로 요약한 핵심 내용을 기반으로, 사용자의 이해도를 객관적으로 점검할 수 있는 맞춤형 연습 문제를 FastAPI 엔드포인트를 통해 비동기적으로 생성합니다. 이는 단순한 정보 습득을 넘어 '진정한 이해'를 돕고, 사용자가 자신의 취약점을 파악하여 메타인지를 높일 수 있도록 설계되었습니다.

나의 역할 및 기여

  • FastAPI (Python) 기반의 비동기 REST API 아키텍처 설계 및 Docker 컨테이너화 주도
  • React.js와 TypeScript를 활용한 반응형 프론트엔드 UI/UX 개발
  • React Query를 통한 전역 서버 상태 관리(캐싱, 에러 처리) 전략 수립 및 적용
  • PDF(PyMuPDF) 및 이미지(Tesseract) 텍스트 추출 모듈 개발 및 LLM API 연동 로직 구현
  • AI 결과물(요약, 문제)의 품질 향상을 위한 프롬프트 엔지니어링 수행 및 최적화
  • GCP Cloud Run 및 Vercel을 활용한 전체 풀스택 애플리케이션 배포

결과 및 성과

  • FastAPI의 비동기 처리와 React Query의 캐싱 전략을 통해, 외부 AI API 호출로 인한 대기 시간에도 불구하고 쾌적한 사용자 경험(UX)을 제공하는 인터페이스 구축
  • Docker와 GCP Cloud Run을 활용한 서버리스 배포를 통해, 트래픽이 없을 때는 비용이 발생하지 않는 효율적인 인프라를 성공적으로 구현
  • 학습 자료의 핵심 내용을 AI가 요약하여, 시험 대비 및 복습에 필요한 시간을 획기적으로 단축시키는 솔루션 구현
  • AI가 생성한 객관적인 문제를 통해 학습자 스스로 메타인지를 높이고 취약점을 보완할 수 있는 자기 주도 학습 환경 제공

회고 (배운 점)

  • FastAPI의 비동기 처리 방식과 의존성 주입(DI)을 실제 프로젝트에 적용하며 백엔드 성능 최적화에 대해 깊이 학습
  • React Query가 비동기 서버 상태 관리에 있어 useState나 Redux 대비 가지는 이점과 캐시 전략의 중요성을 체감
  • Docker를 통해 복잡한 의존성을 가진 환경을 컨테이너화하고 배포하는 풀스택 CI/CD 파이프라인의 중요성을 이해
  • LLM API를 실제 서비스에 연동하고, 목적에 맞는 고품질의 결과를 얻기 위한 프롬프트 엔지니어링의 중요성을 이해

회고 (개선점)

  • 비동기 작업 개선: WebSockets이나 SSE를 도입하여 파일 처리 및 AI 생성 상태를 실시간 스트리밍하여 UX 개선
  • 테스트 코드 도입: Jest, RTL, pytest를 활용한 테스트 코드를 작성하여 코드의 안정성과 신뢰도 확보
  • 대규모 파일 처리: Celery와 RabbitMQ 같은 메시지 큐를 도입하여 무거운 작업을 백그라운드에서 처리하도록 아키텍처 고도화
