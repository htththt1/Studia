"임베디드와 웹, AI를 융합하여 실질적인 문제를 해결하는 개발자가 되고 싶습니다."

기획 배경 및 개발 목표

문제 인식

대학 전공 서적이나 기술 문서는 양이 너무 방대하여, 단순히 읽는 것만으로는 핵심을 파악하기 어렵습니다. 특히 "내가 제대로 이해했나?"를 확인하려면 별도의 문제집이 필요한데, 전공 서적에 딱 맞는 문제는 찾기 어렵다는 점에 주목했습니다.

해결 방안 (Goal)

사용자가 가진 PDF 자료를 기반으로 AI가 즉시 튜터가 되어주는 시스템을 구축했습니다. 핵심 요약과 다양한 유형(객관식/주관식/서술형)의 문제를 자동 생성하여, '읽는 학습'에서 '푸는 학습'으로의 전환을 돕습니다.

시스템 아키텍처

Client (Frontend)
React.js
File Upload / UI
Server (Backend)
FastAPI
REST API Processing
AI Engine
Gemini API
+ OCR / Prompt Eng.
* PDF 텍스트 추출 시 PyMuPDF와 Tesseract(OCR)를 하이브리드로 사용하여 정확도를 높였습니다.

Backend & AI

  • • Python FastAPI (REST API)
  • • Google Gemini API (문제 생성)
  • • PyMuPDF & OCR (텍스트 추출)

Environment

  • • Git & GitHub (형상 관리)
  • • Visual Studio Code
  • • Local Development

서비스 실제 구동 화면

STEP 01

직관적인 파일 업로드

복잡한 설정 없이 PDF 파일을 드래그 앤 드롭하면 AI 분석이 시작됩니다. PyMuPDF를 활용해 대용량 전공 서적도 빠르게 텍스트를 추출합니다.

  • Drag & Drop UI 구현
  • PDF 유효성 검사 및 파일 정보 시각화
STEP 02

AI 맞춤형 문제 풀이

Gemini Pro가 분석한 내용을 바탕으로 객관식, 주관식, 서술형 문제를 생성합니다. 실제 시험과 유사한 UI 환경에서 학습 이해도를 점검할 수 있습니다.

  • 문제 유형 자동 배분 (3:2:1)
  • 실시간 진행률 표시
STEP 03

시각적 학습 분석 리포트

문제 풀이가 끝나면 Chart.js를 활용한 레이더 차트로 나의 강점과 약점을 한눈에 파악할 수 있습니다. 단순 점수가 아닌, 학습 영역별 성취도를 시각화하여 보여줍니다.

  • 영역별 이해도 레이더 차트
  • AI 기반 종합 평가 코멘트
STEP 04

스마트 오답 노트

틀린 문제는 AI의 상세한 해설과 함께 다시 공부할 수 있습니다. 특히 원본 PDF의 해당 페이지를 참조(Reference)로 보여주어, 교재의 어느 부분을 복습해야 할지 정확히 알려줍니다.

  • 틀린 문제 모아보기
  • AI 해설 및 교재 페이지 연동

💡 배운 점 (Keep)

  • 프롬프트 엔지니어링: AI가 JSON 형식으로 정확하게 데이터를 출력하도록 제어하는 방법을 터득했습니다.
  • 하이브리드 데이터 처리: 단순 텍스트 추출뿐만 아니라 OCR을 결합하여 다양한 포맷의 PDF를 처리하는 백엔드 로직을 구현했습니다.
  • Full-Stack 흐름: React와 FastAPI 간의 비동기 통신 및 CORS 처리를 직접 해결하며 웹 서비스 전체 구조를 이해했습니다.

🚀 향후 계획 (Try)

  • 실시간 진행률 표시: 대용량 파일 분석 시 사용자 경험을 위해 WebSocket을 도입하여 진행 상황을 실시간으로 보여주고 싶습니다.
  • 문제 품질 피드백 루프: 사용자가 생성된 문제에 대해 '좋아요/싫어요'를 평가하여 AI 모델을 지속적으로 개선하는 기능을 추가할 예정입니다.
