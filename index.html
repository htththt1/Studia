About Me

전공 서적이나 강의 노트와 같은 방대한 학습 자료를 효율적으로 소화하는 것은 모든 학습자의 난제입니다. 저는 이 문제를 해결하기 위해 '읽는 학습'에서 '푸는 학습'으로의 전환을 돕는 AI 튜터 서비스 Studia를 기획하고 개발했습니다.

이 프로젝트에서 저는 FastAPI 백엔드 구축부터 React 프론트엔드 개발, 그리고 Google Gemini API를 활용한 프롬프트 엔지니어링까지 전 과정을 주도했습니다. 특히 PDF 텍스트 추출의 정확도를 높이기 위해 OCR 기술을 도입하고, AI의 할루시네이션을 제어하며 정형화된 문제 데이터를 생성하는 로직 구현에 집중했습니다.

단순히 기술을 구현하는 것을 넘어, 사용자가 '진짜 이해'를 할 수 있도록 돕는 실용적인 서비스를 만드는 과정에서 풀스택 개발 역량과 AI 응용 능력을 크게 성장시켰습니다.

Project Details

문제 정의 (Problem Definition)

대학 전공 서적이나 기술 문서는 분량이 방대하여 핵심을 파악하는 데 많은 시간이 소요됩니다. 또한, 눈으로만 읽는 수동적인 학습 방식은 메타인지(내가 무엇을 알고 모르는지 파악하는 능력)를 저하시킵니다. "어떻게 하면 방대한 자료를 빠르게 요약하고, 객관적으로 내 이해도를 점검할 수 있을까?"라는 고민에서 Studia가 시작되었습니다.

아키텍처 및 시스템 설계

  • Client-Server 구조: 프론트엔드(React)와 백엔드(FastAPI)를 분리하여 개발 및 유지보수의 효율성을 높였습니다. 두 계층 간 데이터 통신은 RESTful API와 JSON 포맷을 사용합니다.
  • Hybrid Text Extraction (ETL): PDF 처리를 위해 이중 파이프라인을 설계했습니다. 텍스트 레이어가 있는 PDF는 PyMuPDF로 고속 추출하고, 스캔본과 같은 이미지형 PDF는 Tesseract OCR 엔진으로 자동 전환하여 처리함으로써 데이터 유실을 방지했습니다.
  • AI Pipeline: 추출된 텍스트를 Google Gemini 2.5 Flash 모델에 전달합니다. 이때 프롬프트 엔지니어링을 통해 '전문 튜터' 페르소나를 부여하고, 출력 형식을 엄격한 JSON 스키마로 제어하여 프론트엔드에서의 파싱 오류를 차단했습니다.

핵심 기능 (Core Features)

AI 요약 및 3:2:1 문제 생성

단순한 문제 생성이 아닙니다. 학습 효율이 가장 높은 객관식(3) : 주관식(2) : 서술형(1) 비율로 문제를 자동 출제합니다. 동시에 전체 문서의 핵심 내용을 요약하여 제공함으로써 학습 전반의 맥락을 잡을 수 있게 돕습니다.

5단계 맞춤형 피드백 시스템

사용자의 점수에 따라 100점부터 70점 미만까지 5단계로 세분화된 피드백을 제공합니다. 단순한 점수 통보가 아니라, 현재 학습 상태에 맞는 격려와 조언을 통해 학습 동기를 부여합니다.

시각적 분석 리포트

Chart.js를 활용한 레이더 차트로 학습자의 이해도, 응용력, 분석력 등 5가지 지표를 시각화하여 보여줍니다. 또한 '오답 노트' 기능을 통해 틀린 문제만 다시 풀 수 있는 반복 학습 환경을 제공합니다.

나의 역할 및 기여 (100%)

  • Full Stack Development: React 기반의 SPA 프론트엔드와 FastAPI 기반의 REST API 백엔드 전체 구현
  • Data Processing: PyMuPDF와 Tesseract OCR을 결합한 하이브리드 텍스트 추출 로직 개발
  • Prompt Engineering: Gemini API가 JSON 포맷을 준수하고, 특정 비율(3:2:1)로 문제를 출제하도록 프롬프트 최적화
  • UI/UX Design: Tailwind CSS를 활용한 반응형 대시보드 및 직관적인 퀴즈 인터페이스 설계
  • System Integration: 프론트엔드와 백엔드 간의 CORS 설정 및 비동기 파일 업로드/데이터 처리 연동

회고 (배운 점)

  • LLM을 애플리케이션에 통합할 때, 정형 데이터(JSON) 출력을 유도하는 프롬프트 설계의 중요성을 체감했습니다.
  • OCR 엔진(Tesseract)을 서버에 통합하며, 외부 의존성 라이브러리를 관리하는 백엔드 환경 구성 능력을 키웠습니다.
  • React의 상태 관리(useState)와 비동기 통신(Fetch API)을 통해 복잡한 데이터 흐름을 제어하는 법을 익혔습니다.

회고 (개선점)

  • 대용량 PDF 처리 시 사용자 대기 시간을 줄이기 위해 WebSocket을 도입하여 실시간 진행률을 보여주고 싶습니다.
  • 생성된 문제의 퀄리티를 사용자가 평가(좋아요/싫어요)하여 AI 프롬프트를 지속적으로 개선하는 피드백 루프를 추가할 예정입니다.
